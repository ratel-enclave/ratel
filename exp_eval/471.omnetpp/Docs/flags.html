471.omnetpp Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

Sections

Selecting one of the following will take you directly to that section:

  • Portability Flags
  • Mandatory Flags
  • Forbidden Flags

Portability Flags

  • -DSPEC_CPU_NO_NESTED_CLASS_ACCESS
  • -DSPEC_CPU_STDLIB_USING_ERROR
  • -DSPEC_CPU_WIN64
    • -DSPEC_CPU_NO_NESTED_CLASS_ACCESS
    • (?:^|(?<=\s))[-/]DSPEC_CPU_NO_NESTED_CLASS_ACCESS(?:=\S*)?(?=\s|$)

    • SPEC_CPU_NO_NESTED_CLASS_ACCESS can be used for compilers that do not allow access to the private members of a nested class.

    • -DSPEC_CPU_STDLIB_USING_ERROR
    • (?:^|(?<=\s))[-/]DSPEC_CPU_STDLIB_USING_ERROR(?:=\S*)?(?=\s|$)
    • This macro can be used by compilers that need "std::" to access functions in cmath.
    • -DSPEC_CPU_WIN64
    • (?:^|(?<=\s))[-/]DSPEC_CPU_WIN64(?:=\S*)?(?=\s|$)

    • This macro indicates that the build is for 64-bit Windows system.

Mandatory Flags

  • -I., -Iomnet_include, -Ilibs/envir
    • -I., -Iomnet_include, -Ilibs/envir
    • -I(?:\.|omnet_include|libs/envir)(?:\s+|$)

    • This flag is set for all builds in Spec/object.pm.
